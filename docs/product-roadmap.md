# WhyThough Product Roadmap

WhyThough should feel like a real small product, not a demo wrapped around a database. The core experience is simple: choose a category, answer a set of strange questions, get a result that feels oddly personal, then come back later to compare old attempts.

## Design Direction

The base interface should be calm, minimal, and Apple-inspired:

- light surfaces
- soft shadows
- clear hierarchy
- restrained animation
- direct labels
- generous spacing
- no decorative visual noise

Each category should still keep its own identity. The app shell stays quiet; the category cards, quiz panels, result accents, and small atmospheric details carry the personality.

## Category Identities

- Midnight Convenience Store: cool light, late-night blue, quiet retail loneliness
- Human Confusing Behavior: paper-white interface, red observation marks, everyday behavior notes
- Cosmic Nonsense: deep violet accents, experimental lab feeling, absurd philosophy
- Internet Mental State: blue screen glow, tabs, notifications, late-night scrolling energy
- The System Is Watching You: stark monochrome, red warnings, controlled tension

## Product Milestones

### 1. Make It Feel Real

- Keep the project language focused on the product and its users.
- Reduce AI-like over-explanation.
- Keep the humor, but make buttons and navigation direct.
- Make the first screen immediately understandable.

### 2. Improve The Quiz Flow

- Show one question at a time.
- Lock answers after selection.
- Add clearer progress feedback.
- Make result pages shorter and easier to share.

### 3. Strengthen Content Tools

- Improve admin question editing.
- Add category metadata management.
- Add draft/published states for question sets.
- Add safer import validation and preview before saving.

### 4. Make Results Shareable

- Generate a compact result card.
- Add share/download actions.
- Let users compare attempts over time.

### 5. Prepare For Deployment

- Add production environment documentation.
- Add seed data instructions.
- Add basic smoke tests.
- Deploy frontend and backend.

## Voice Principles

- Be funny by being precise.
- Use absurdity where it lands, not everywhere.
- Keep navigation plain.
- Keep result writing human and specific.
- Avoid sounding like a chatbot pretending to be mysterious.
